## 4. 🔄 วิธีการใช้งานและกระบวนการทำงาน (Workflows)

### 👤 4.1 กระบวนการทำงานของผู้รับบริการ/ประชาชน (User Workflow)
1. **เลือกประเภทบริการ:** เข้าสู่เว็บไซต์ [step-blueprint-portal-e2b08.web.app](https://step-blueprint-portal-e2b08.web.app) เลือกแท็บที่ต้องการระหว่าง **"ยื่นขอตรวจสอบแบบแปลน"** หรือ **"ยื่นขอคัดลอกแบบแปลน"**
2. **กรอกข้อมูลและแนบไฟล์:** กรอกข้อมูลผู้ยื่น ประเภทงาน และอัปโหลดไฟล์แบบแปลน (PDF ขนาดไม่เกิน 20 MB) แล้วกดส่งข้อมูล
3. **รับรหัสติดตามและอีเมลยืนยัน:** ระบบจะสร้างรหัสติดตามเรื่อง (เช่น `REQ-1001`) และส่งอีเมลยืนยันการรับเรื่องฉบับแรกไปที่อีเมลของผู้ยื่นทันที
4. **ตรวจสอบสถานะ Real-time:** ผู้ใช้สามารถนำรหัส `REQ-XXXX` ไปกรอกที่แท็บ **"ตรวจสอบสถานะคำร้อง"** เพื่อดูผลการพิจารณา ความเห็นวิศวกร และชื่ออีเมลของเจ้าหน้าที่ผู้รับผิดชอบงานนั้นๆ ได้ตลอด 24 ชั่วโมง

### 🛡️ 4.2 กระบวนการทำงานของเจ้าหน้าที่ผู้ดูแลระบบ (Admin Workflow)
1. **เข้าสู่ระบบที่ปลอดภัย:** คลิกปุ่ม "สำหรับเจ้าหน้าที่" ที่มุมขวาบน ระบุอีเมล Admin ระบบจะส่ง Magic Link ไปที่กล่องจดหมาย ให้คลิกลิงก์เพื่อเข้าสู่ระบบหลังบ้าน
2. **เข้าถึงเมนูพิเศษ:** เมื่อเข้าสู่ระบบสำเร็จ เมนู **"ระบบหลังบ้าน (Admin Dashboard)"** และ **"ฐานความรู้กฎหมาย"** จะปรากฏขึ้นบน Navbar โดยอัตโนมัติ
3. **ตรวจสอบคำร้องและไฟล์แปลน:** ดูตารางรายการคำร้องทั้งหมด สามารถกดปุ่ม "📄 ดูเอกสาร" เพื่อเปิดดูไฟล์ PDF บน Google Drive
4. **ใช้ AI ตรวจแบบแปลน (AI Review):** กดปุ่ม **"🤖 AI ตรวจแบบ"** ระบบจะเรียก Gemini 2.5 Flash ให้ตรวจสอบข้อมูลคำร้องเทียบกับ พ.ร.บ. อาคาร และกฎกระทรวง พ.ศ. 2566 ทั้ง 3 ฉบับ (วัสดุก่อสร้าง, ฐานราก, โครงสร้าง) พร้อมสรุปผลให้อัตโนมัติ
5. **ร่างคำตอบกลับทางการ (AI Autofill):** กดปุ่ม **"✍️ ร่างคำตอบกลับ"** เลือกผลการตัดสินใจ (*อนุมัติ / ขอแก้ไขรายละเอียด / ปฏิเสธ*) และระบุข้อสังเกต กดปุ่มให้ AI ร่างข้อความอีเมลภาษาไทยที่เป็นทางการ สุภาพ และชัดเจนให้ทันที
6. **ส่งอีเมลและแยกระบบติดตามรายบุคคล (Smart Email Routing):** เมื่อกดปุ่มส่งอีเมล ระบบจะส่งข้อความต่อท้ายกระทู้เดิม (Gmail Threading) พร้อมกำหนด Header `Reply-To` และ `Cc` เป็นอีเมลของเจ้าหน้าที่ผู้ตอบ (เช่น `kampanat@step.cmu.ac.th`) ทำให้เมื่อผู้ยื่นเรื่องกดตอบกลับ (Reply) ข้อความจะส่งตรงเข้ากล่องจดหมายส่วนตัวของเจ้าหน้าที่ผู้ดูแลงานนั้นทันที พร้อมบันทึกชื่อผู้ตอบลงในฐานข้อมูล Google Sheet (คอลัมน์ N)

## 5. 🔑 ระบบความปลอดภัยและสิทธิ์การเข้าถึง (Token & Security System)

### 🔒 5.1 Passwordless Magic Link (JWT Token Auth)
- **การทำงาน:** ระบบไม่ใช้รหัสผ่าน (No Password Required) เพื่อป้องกันปัญหาการเดารหัสหรือรหัสผ่านรั่วไหล
- **ความปลอดภัย:** เมื่อระบุอีเมล ระบบจะสร้าง **JSON Web Token (JWT)** ที่เข้ารหัสลับด้วย `JWT_SECRET` ฝังไว้ใน Magic Link และส่งไปทางอีเมล ลิงก์นี้มีอายุการใช้งาน 24 ชั่วโมง
- **การตรวจสอบสิทธิ์ (Middleware):** ทุกครั้งที่เรียกใช้งาน API หลังบ้าน (เช่น ดูข้อมูล, สั่ง AI ตรวจแบบ, ส่งอีเมล) ระบบจะตรวจจับและยืนยันตัวตนจาก JWT Token เสมอ หาก Token หมดอายุหรือไม่ถูกต้อง ระบบจะปฏิเสธการเข้าถึงทันที

### 🛡️ 5.2 Google OAuth 2.0 (Refresh Token System)
- **การทำงาน:** ระบบเชื่อมต่อกับบริการ Google (Sheets API, Drive API, Gmail API) ผ่านโปรโตคอล OAuth 2.0 โดยใช้ `CLIENT_ID`, `CLIENT_SECRET` และ `REFRESH_TOKEN`
- **ความปลอดภัย:** ทำให้เซิร์ฟเวอร์สามารถอ่าน/เขียนข้อมูลใน Spreadsheet และส่งอีเมลได้โดยไม่ต้องเก็บรหัสผ่านจริงของบัญชี Google ปลอดภัยตามมาตรฐานสากล

## 6. 🤖 ระบบ AI ข้อจำกัดการใช้งาน และนโยบายห้ามเสียเงิน (AI Model & Quota Limits)

### 🧠 6.1 รุ่นของ AI และสถาปัตยกรรม (AI Model)
- **Model:** Google Gemini 2.5 Flash (ผ่าน Google Generative AI REST API)
- **การทำงาน:** ประมวลผลแบบ Real-time โดยดึงข้อกำหนดจากฐานความรู้ (Knowledge Base) มาวิเคราะห์ร่วมกับข้อมูลคำร้อง

### ⏳ 6.2 ข้อจำกัดโควต้าและการรีเซ็ตเวลา (Quota & Reset Times)
- **ระบบทำงานภายใต้ Free Tier 100%:** ไม่มีค่าใช้จ่ายในการประมวลผล
- **Rate Limit ต่อนาที:** หากมีการกดใช้งาน AI ถี่เกินไป (เกินลิมิตต่อนาที) ระบบจะแจ้งเตือน ให้รอประมาณ **1 - 2 นาที** แล้วกดใช้งานใหม่ได้ทันที
- **Daily Limit รายวัน:** หากใช้งานจนครบโควต้าฟรีประจำวัน ระบบจะหยุดให้บริการ AI ชั่วคราว โดยโควต้าจะทำการรีเซ็ตใหม่ทุกวันเวลา **14:00 น. ตามเวลาประเทศไทย** (ตรงกับเวลาเที่ยงคืนมาตรฐานแปซิฟิก Pacific Time ของ Google)
- **ระบบป้องกันและแจ้งเตือน (Quota Guard):** เมื่อติดลิมิต หน้าเว็บจะแสดงกล่องแจ้งเตือนสีแดงทันที พร้อมระบุเวลาการรีเซ็ตอย่างชัดเจน

### 🚫 6.3 นโยบายห้ามใช้ระบบเสียเงินโดยเด็ดขาด (Strict No-Paid-Tier Policy)
> ⚠️ **คำสั่งการสำคัญที่สุด:** ห้ามทำการผูกบัตรเครดิต อัปเกรดบัญชี Google Cloud / Google AI Studio เป็นแบบเสียเงิน (Paid Tier / Pay-as-you-go) หรือเปิดใช้งาน Billing สำหรับ Gemini API โดยเด็ดขาด โครงการนี้ถูกออกแบบมาให้บริหารจัดการภายใต้โควต้าฟรี (Free Tier) เท่านั้น หากโควต้าหมดให้รอการรีเซ็ตตามเวลาที่กำหนดครับ

4. **ขนาดไฟล์ที่รองรับ:** ระบบรองรับการรับไฟล์แบบแปลนผ่านลิมิตของ Google Drive (แนะนำขนาดไฟล์ PDF ไม่เกิน 20 MB ต่อรายการเพื่อความรวดเร็วในการอัปโหลด)

เพื่อให้ระบบทำงานร่วมกันระหว่างหน้าเว็บและเซิร์ฟเวอร์หลังบ้านโดยไม่เกิดข้อผิดพลาด และไม่มีค่าใช้จ่ายตลอดอายุการใช้งาน ให้ปฏิบัติตามขั้นตอนการตั้งค่าดังนี้:

### 🚀 8.1 การนำโค้ดขึ้น GitHub และ Render.com (Backend Setup)
1. **เตรียมไฟล์ขึ้น GitHub:** เลือกอัปโหลดเฉพาะโฟลเดอร์และไฟล์ที่จำเป็น ได้แก่ โฟลเดอร์ `functions`, โฟลเดอร์ `public`, และไฟล์ `package.json`, `render.yaml`, `firebase.json`, `google_assets_config.json`, `google_links.md` (ไม่ต้องอัปโหลดโฟลเดอร์ `node_modules` และไฟล์ `.env` ขึ้น GitHub เพื่อความปลอดภัย)
2. **สร้าง Blueprint Service บน Render.com:**
   - เข้าสู่ระบบ [Render.com](https://render.com) ด้วยบัญชี GitHub
   - กดปุ่ม **"+ New" -> "Blueprint"** เลือก Repository ของโครงการ
   - ระบบจะอ่านไฟล์ `render.yaml` และสร้างบริการ Web Service ชื่อ `step-blueprint-portal-backend` บนแพ็กเกจ **Free Tier (ฟรี 100%)** ให้อัตโนมัติ

### 🔐 8.2 การตั้งค่ารหัสลับ Environment Variables (Secret Keys Setup)
เนื่องจากไฟล์ `.env` ถูกตัดออกจากการอัปโหลดขึ้น GitHub เพื่อความปลอดภัยของรหัสลับ เซิร์ฟเวอร์บน Render.com จึงต้องตั้งค่ารหัสลับผ่านหน้าเว็บ Dashboard ดังนี้:
1. เข้าไปที่เมนู **"Environment"** ของโปรเจกต์ `step-blueprint-portal-backend` บนเว็บ Render
2. กดปุ่ม **"Edit"** ที่มุมขวาบนของกรอบ Environment Variables แล้วกดปุ่ม **"Add from .env"**
3. ก๊อปปี้รหัสลับทั้ง 4 ตัวไปวางลงในช่องว่าง:
```env
CLIENT_ID=<YOUR_CLIENT_ID>
CLIENT_SECRET=<YOUR_CLIENT_SECRET>
REFRESH_TOKEN=<YOUR_REFRESH_TOKEN>
GEMINI_API_KEY=<YOUR_GEMINI_API_KEY>
```
4. กดปุ่ม **"Save Changes"** เซิร์ฟเวอร์จะรีสตาร์ทตัวเองและพร้อมเชื่อมต่อกับ Google Sheets, Google Drive, Gmail และ AI Gemini ทันที

### 🌐 8.3 การเชื่อมต่อกับหน้าเว็บ Firebase Hosting (Frontend Setup)
1. ในไฟล์ `public/app.js` ได้กำหนดค่า URL ของ Render ไว้ที่ตัวแปร `RENDER_BACKEND_URL`:
```javascript
const RENDER_BACKEND_URL = 'https://step-blueprint-portal-backend.onrender.com';
```
2. เมื่อทำการ Deploy หน้าเว็บด้วยคำสั่ง `firebase deploy --only hosting` หน้าเว็บที่อยู่บน Firebase Hosting (`https://step-blueprint-portal-e2b08.web.app`) จะทำการเรียกใช้งาน API และ AI ผ่าน Render Backend URL นี้โดยอัตโนมัติ ทำให้ระบบทั้งสองฝั่งทำงานร่วมกันได้อย่างสมบูรณ์แบบครับ
